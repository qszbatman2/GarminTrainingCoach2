# Docs Index

文档按用途归档，新增文档优先放入下面目录。

```text
docs/
  architecture/  架构方案和核心技术设计
  integration/   外部系统集成说明
  ops/           部署、同步、定时任务和运维策略
  plans/         已执行或待执行的阶段性开发计划
  product/       产品页面、交互和展示逻辑说明
  rules/         训练分析规则和业务判断边界
```

约定：

- 长期有效的系统设计放 `architecture/`。
- 和部署、数据同步、线上排障相关的内容放 `ops/`。
- 训练决策底线、周训练量判断等业务规则放 `rules/`。
- 临时计划可以先放 `plans/`，落地后再沉淀到对应长期目录。
